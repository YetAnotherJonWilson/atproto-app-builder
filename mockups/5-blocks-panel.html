Mockup 5: Blocks Panel — Chip-based Requirement Selection

What this mockup shows

  • Scene A: Blocks panel with existing blocks (desktop) — block cards at top, unassigned requirements list below with quick-create shortcuts (hover to reveal "+ Block" button with name dropdown). Unassigned section disappears when all requirements are assigned.
  • Scene B: "New Block" form open — name field + chip-based requirement selector. Selected requirements appear as numbered chips above the available list.
  • Scene C: Narrow/mobile accordion view — same blocks panel content inside an accordion section
