THE APP BUILDER 

In the following steps, we'll build a decentralized web app, which is a web page or application where:

  1. You login with an identity that you own
  2. The app uses data from a source that you own

Some examples are:

  • A meditation app with a timer and sound settings, that tracks your sessions.
  • A grocery list app that you might share with your family or roommates.
  • An event-planning app that helps with organization, tracking RSVP's, schedules, and tasks.

Using the AT Protocol, you can:

  1. Login/logout:
    This step will be mostly handled for you by existing libraries.
  2. Retrieve data from one or more Personal Data Servers:
    Think about what data you need in your app, and how you will define it. In ATProtocol applications, your data exists as Records and Properties on those records. For example, an item on a todo list might be a Record of type "todo", with Properties like "text" (the content of the todo item), and "completed" (a boolean indicating whether the item is done).
  3. Use retrieved data to create a user interface (UI):
    Determine how you will display your dataâ€”it could be displayed directly (e.g. a todo list that shows every item by default), or used to create a list of options (e.g. a dropdown menu that allows you to select an option). For now, the app-builder tool will create default views and a simple UI for you. In the future, our goal is to allow you to customize the UI further.
  4. Interact with your data:
    Based on your input about what methods should be available to interact with your data (creating, retrieving, updating, and/or deleting records), the app-builder tool will generate the necessary methods to enable those interactions.
